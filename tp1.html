Introduction

Apache Hadoop est un framework open-source conçu pour le stockage et le traitement distribué de grands ensembles de données (Big Data) sur des clusters de machines standard. Il repose sur quatre modules principaux :

  • Hadoop Common : Les utilitaires communs supportant les autres modules.
  • HDFS (Hadoop Distributed File System) : Un système de fichiers distribué offrant un accès haut débit aux données applicatives.
  • Hadoop YARN : Un framework pour la planification des tâches et la gestion des ressources du cluster.
  • Hadoop MapReduce : Un système basé sur YARN pour le traitement parallèle de grands ensembles de données.
Objectif du TP : Ce guide détaille l'installation pas à pas d'Apache Hadoop version 3.4.3 sur le système d'exploitation Ubuntu 22.04 LTS. Nous utiliserons le Mode Pseudo-Distribué.

Note sur le mode Pseudo-Distribué : Dans ce mode, tous les démons Hadoop (NameNode, DataNode, ResourceManager, NodeManager) s'exécutent sur une seule machine (votre ordinateur), chacun dans un processus Java distinct. Cela simule un cluster complet et est idéal pour l'apprentissage, le développement et le test.

1. Prérequis

1.1 Configuration matérielle recommandée

Composant Configuration Minimale Recommandée
Système d'exploitation Ubuntu 22.04 LTS (64-bit) Ubuntu 22.04 LTS (64-bit)
Mémoire RAM 4 Go 8 Go ou plus
Espace Disque 20 Go libres 50 Go libres (SSD recommandé)
Privilèges Accès root (sudo) Accès root (sudo)

1.2 Mise à jour du système

Avant de commencer, assurez-vous que votre système est à jour :

sudo apt update

2. Installation de Java OpenJDK 21

Depuis la version 3.4.x, Hadoop supporte officiellement les versions récentes de Java LTS. Nous utiliserons OpenJDK 21 Java 21 pour ce TP. 

sudo apt install openjdk-21-jdk -y

Vérifiez l'installation :

java -version

Résultat attendu (similaire à) : openjdk version "21.0.4" ...

Identifiez le chemin d'installation de Java pour la configuration future :

which java
readlink -f $(which java)

Le chemin devrait ressembler à : /usr/lib/jvm/java-21-openjdk-amd64/bin/java.
Le JAVA_HOME sera donc : /usr/lib/jvm/java-21-openjdk-amd64.

3. Configuration SSH et utilisateur Hadoop

Hadoop nécessite SSH pour gérer ses nœuds, même en mode pseudo-distribué.

3.1 Installation SSH et pdsh

sudo apt install openssh-server openssh-client pdsh -y

3.2 Création de l'utilisateur hadoop

Il est recommandé de créer un utilisateur système dédié pour isoler l'installation Hadoop.

sudo useradd -m -s /bin/bash hadoop
sudo passwd hadoop
# Entrez un mot de passe simple (ex: hadoop) pour le TP
sudo usermod -aG sudo hadoop

Connectez-vous avec ce nouvel utilisateur :

su - hadoop

3.3 Configuration SSH sans mot de passe

Hadoop doit pouvoir se connecter à localhost sans intervention humaine.

# Générer une paire de clés RSA (laissez la passphrase vide en appuyant sur Entrée)
ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa

# Autoriser la clé publique
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
chmod 0600 ~/.ssh/authorized_keys

# Tester la connexion
ssh localhost
Important : Lors de la première connexion ssh localhost, tapez "yes" pour accepter l'empreinte. Si vous êtes connecté sans demande de mot de passe, la configuration est réussie. Tapez exit pour revenir au shell normal si nécessaire.

4. Téléchargement et installation de Hadoop 3.4.3

Nous allons télécharger la version binaire stable depuis les miroirs Apache.

# Téléchargement
wget https://dlcdn.apache.org/hadoop/common/hadoop-3.4.3/hadoop-3.4.3.tar.gz

# Extraction
tar -xvzf hadoop-3.4.3.tar.gz

# Déplacement vers /opt 
sudo mv hadoop-3.4.3 /opt/hadoop

# Attribution des droits à l'utilisateur hadoop
sudo chown -R hadoop:hadoop /opt/hadoop

Vérification rapide :

ls -l /opt/hadoop

5. Configuration des variables d'environnement

Il faut configurer le shell pour qu'il trouve les commandes Hadoop et Java.

Éditez le fichier .bashrc de l'utilisateur hadoop :

nano ~/.bashrc

Ajoutez les lignes suivantes à la fin du fichier :

# ---- Hadoop Environment Variables ----
export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64
export HADOOP_HOME=/opt/hadoop
export HADOOP_INSTALL=$HADOOP_HOME
export HADOOP_MAPRED_HOME=$HADOOP_HOME
export HADOOP_COMMON_HOME=$HADOOP_HOME
export HADOOP_HDFS_HOME=$HADOOP_HOME
export HADOOP_YARN_HOME=$HADOOP_HOME
export HADOOP_COMMON_LIB_NATIVE_DIR=$HADOOP_HOME/lib/native
export PATH=$PATH:$HADOOP_HOME/sbin:$HADOOP_HOME/bin
export HADOOP_OPTS="-Djava.library.path=$HADOOP_HOME/lib/native"

Sauvegardez (Ctrl+O, Entrée) et quittez (Ctrl+X). Appliquez les changements :

source ~/.bashrc

Testez si Hadoop est reconnu :

hadoop version

Vous devriez voir : Hadoop 3.4.3

Enfin, éditez le fichier de configuration interne hadoop-env.sh pour forcer le chemin Java :

nano $HADOOP_HOME/etc/hadoop/hadoop-env.sh

Trouvez la ligne export JAVA_HOME=... (souvent commentée) et remplacez-la par :

export JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64

6. Configuration du cluster Hadoop (Mode Pseudo-Distribué)

Nous devons modifier 4 fichiers XML situés dans $HADOOP_HOME/etc/hadoop/.

6.1 Création des répertoires de données HDFS

sudo mkdir -p /home/hadoop/hdfs/namenode
sudo mkdir -p /home/hadoop/hdfs/datanode
sudo chown -R hadoop:hadoop /home/hadoop/hdfs/

6.2 Fichier core-site.xml

Configure l'adresse du NameNode (le maître du système de fichiers).

nano $HADOOP_HOME/etc/hadoop/core-site.xml

<configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:8020</value>
    </property>
</configuration>

6.3 Fichier hdfs-site.xml

Configure la réplication (1 seule copie car 1 seul nœud) et les chemins de stockage.

nano $HADOOP_HOME/etc/hadoop/hdfs-site.xml

<configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
    <property>
        <name>dfs.name.dir</name>
        <value>file:///home/hadoop/hdfs/namenode</value>
    </property>
    <property>
        <name>dfs.data.dir</name>
        <value>file:///home/hadoop/hdfs/datanode</value>
    </property>
</configuration>

6.4 Fichier mapred-site.xml

Indique à MapReduce d'utiliser YARN comme gestionnaire de ressources.

nano $HADOOP_HOME/etc/hadoop/mapred-site.xml

<configuration>
    <property>
        <name>mapreduce.framework.name</name>
        <value>yarn</value>
    </property>
    <property>
        <name>mapreduce.application.classpath</name>
        <value>$HADOOP_MAPRED_HOME/share/hadoop/mapreduce/*:$HADOOP_MAPRED_HOME/share/hadoop/mapreduce/lib/*</value>
    </property>
</configuration>

6.5 Fichier yarn-site.xml

Active le service de shuffle nécessaire pour MapReduce.

nano $HADOOP_HOME/etc/hadoop/yarn-site.xml

<configuration>
    <property>
        <name>yarn.nodemanager.aux-services</name>
        <value>mapreduce_shuffle</value>
    </property>
    <property>
        <name>yarn.nodemanager.env-whitelist</name>
        <value>JAVA_HOME,HADOOP_COMMON_HOME,HADOOP_HDFS_HOME,HADOOP_CONF_DIR,CLASSPATH_PREPEND_DISTCACHE,HADOOP_YARN_HOME,HADOOP_HOME,PATH,LANG,TZ,HADOOP_MAPRED_HOME</value>
    </property>
</configuration>

7. Initialisation et démarrage du cluster

7.1 Formatage du NameNode

Cette étape initialise le système de fichiers HDFS. Elle ne doit être faite qu'une seule fois.

hdfs namenode -format
ATTENTION : Ne formatez le NameNode qu'une seule fois lors de l'installation ! Si vous reformatez un NameNode en cours d'utilisation, vous perdrez toutes les données stockées dans HDFS et casserez la synchronisation avec les DataNodes.

7.2 Démarrage des services

Avant de démarrer, définissons le type de connexion RCMD pour éviter une erreur courante avec pdsh :

export PDSH_RCMD_TYPE=ssh

Vous pouvez démarrer tous les services (HDFS + YARN) en une seule commande :

start-all.sh

Ou séparément :

start-dfs.sh
start-yarn.sh

7.3 Arrêt des services

Pour arrêter proprement le cluster à la fin du TP :

stop-all.sh

8. Vérification et validation

8.1 Vérification des processus Java actifs

Utilisez l'outil jps (Java Process Status) pour vérifier que tous les démons tournent.

jps

Vous devez voir impérativement ces 6 processus (les numéros PID seront différents) :

Nom du Processus Rôle
NameNode Maître HDFS (Gestion des métadonnées)
DataNode Esclave HDFS (Stockage des blocs de données)
ResourceManager Maître YARN (Gestion globale des ressources)
NodeManager Esclave YARN (Exécution des conteneurs)
SecondaryNameNode Maintenance des métadonnées (Checkpoints)
Jps L'outil de vérification lui-même

8.2 Interfaces Web de supervision

Ouvrez votre navigateur (Firefox/Chrome) aux adresses suivantes :

  • HDFS NameNode UI : http://localhost:9870 (Vue d'ensemble du stockage)
  • YARN ResourceManager UI : http://localhost:8088 (Vue des jobs en cours)
  • HDFS DataNode UI : http://localhost:9864

8.3 Tests fonctionnels HDFS

Testons l'écriture et la lecture sur le système de fichiers distribué.

# 1. Créer des répertoires utilisateurs dans HDFS
hdfs dfs -mkdir -p /user/hadoop
hdfs dfs -mkdir /test_input

# 2. Copier un fichier local (ex: logs système) vers HDFS
hdfs dfs -put /var/log/syslog /test_input/

# 3. Lister le contenu du répertoire HDFS
hdfs dfs -ls /test_input/

# 4. Lire le début du fichier depuis HDFS
hdfs dfs -cat /test_input/syslog | head -5

8.4 Exécution d'un job MapReduce (Exemple WordCount/Grep)

Nous allons lancer un programme d'exemple (Grep) qui compte les occurrences d'un motif.

# Préparation : Copier les fichiers XML de config comme données d'entrée
hdfs dfs -mkdir input
hdfs dfs -put $HADOOP_HOME/etc/hadoop/*.xml input

# Exécution du job MapReduce
hadoop jar $HADOOP_HOME/share/hadoop/mapreduce/hadoop-mapreduce-examples-3.4.3.jar grep input output 'dfs[a-z.]+'

# Affichage des résultats
hdfs dfs -cat output/*
Si vous voyez une liste de propriétés commençant par "dfs..." avec un nombre à côté, félicitations ! Votre cluster Hadoop est fonctionnel capable d'exécuter des traitements MapReduce.

9. Résolution des problèmes courants (Dépannage)

Symptôme Cause probable Solution
Connection refused sur localhost:8020 NameNode n'est pas démarré Vérifiez les logs dans $HADOOP_HOME/logs/. Avez-vous formaté le NameNode ?
NameNode est en safe mode Protection automatique au démarrage Attendre ou forcer la sortie : hdfs dfsadmin -safemode leave
Permission denied lors du start-all.sh PDSH tente d'utiliser rsh au lieu de ssh Exécutez : export PDSH_RCMD_TYPE=ssh
SSH demande un mot de passe Clés SSH mal configurées ou droits incorrects Vérifiez les droits : chmod 600 ~/.ssh/authorized_keys et chmod 700 ~/.ssh
DataNode absent dans JPS Incompatibilité ClusterID après reformatage Supprimez les données : rm -rf /home/hadoop/hdfs/datanode/* et redémarrez.
